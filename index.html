Uniform Resource Locator Info

Background

The line displayed by web browsers above the page in an address bar is a 'Uniform Resource Locator' or URL. The specification for a URL is laid down in rfc-1738.
Most common URLs are made up of these components: 

  scheme           host         port             path          query                         

 >>-http://--+-host name--+--+--------+--/--path component--?-query--#fragment--
             '-IP address-'  '-:-port-'

  • Scheme: identifies the protocol to be used to access the resource on the Internet. For instance 'http', 'https', 'ftp' etc.
  • Host (or IP address): identifies the host that holds the resource. For example, www.example.com. For one hostname there may be many servers, or vice versa one server may provide services for many hostnames.
  • Port number: is optional, will default to common values like 80 for http.
  • Path: identifies the specific resource in the host that the web client wants to access. For example, /software/index.html.
  • Query string and fragments: if used, it follows the path component, and provides a string of information that the resource can use for some purpose.
    For example tell 'Bing' to look for images of 'panda': http://www.bing.com/images/search?q=panda
